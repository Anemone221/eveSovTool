# Structures

## Purpose
Tree listing (region → constellation → system → cards) of all player structures within a plan. Supports manual add via dropdown, clipboard import from EVE's Show Info export, and auto-generation of Ansiblex cards when the ALN upgrade is assigned. Shows Metenox/Athanor/Tatara profitability when moon scan data and market prices are both available.

## Schema

```sql
CREATE TABLE IF NOT EXISTS plan_structures (
  id             INTEGER PRIMARY KEY AUTOINCREMENT,
  plan_id        INTEGER NOT NULL REFERENCES plans(id) ON DELETE CASCADE,
  system_id      INTEGER NOT NULL REFERENCES systems(id),
  structure_type TEXT NOT NULL,
  name           TEXT,
  location       TEXT,
  moon_id        INTEGER,
  notes          TEXT,
  source         TEXT NOT NULL DEFAULT 'manual'
);
CREATE INDEX IF NOT EXISTS idx_plan_structures_system
  ON plan_structures(plan_id, system_id);
```

`structure_type` values: `'Ansiblex'`, `'Metenox'`, `'Athanor'`, `'Tatara'`, `'Sotiyo'`, `'Other'`.
`location` values: `'Deep'`, `'Planet'`, `'Moon'`, `'Gate'`, `'Ansiblex'`.
`source` values: `'manual'`, `'clipboard'`, `'upgrade'` (auto-generated by ALN assignment).

## IPC

- `structures.list(planId, systemId?)` → `StructureNode[]` — grouped tree; omit `systemId` for plan-wide view.
- `structures.add(planId, systemId, structure)` → `{ id }` — inserts a row; broadcasts `plan-changed`.
- `structures.remove(planId, structureId)` — deletes; broadcasts `plan-changed`.
- `structures.importClipboard(planId, systemId, text)` → `{ count }` — parses EVE clipboard text, inserts rows; broadcasts `plan-changed`.
- `structures.profitability(structureId)` → `ProfitabilityResult | null` — null if moon data or market prices are absent.

## Key decisions

- **Clipboard import UI**: a `<textarea>` inline form (no `window.prompt()`). The parser runs in the main process IPC handler so it has access to the `systems` table for name→id lookup.
- **ALN auto-card**: when `plans.assignUpgrade` assigns an ALN upgrade, it also calls the equivalent of `structures.add` with `source: 'upgrade'` and `structure_type: 'Ansiblex'`. When ALN is removed, the auto-generated card is also deleted (identified by `source = 'upgrade'`).
- **Sotiyo flag**: in `SystemDetail`, if `structures.list(planId, systemId)` contains a `'Sotiyo'` entry AND the assigned upgrades include the Supercapital Production upgrade, a flag class is added to the system title. The flag propagates to the Universe tree via the existing `plan-changed` event.
- **Profitability**: gated on `data.hasMarketData()`. If unavailable, the profitability cell shows "Enable Data Sync." The calculation uses moon ore composition from `moon_scans` × ore prices from the market data table.
- The Structures page defaults to showing the active plan's structures. Passing `planId = null` would show all plans — not implemented initially; add if users need cross-plan views.
